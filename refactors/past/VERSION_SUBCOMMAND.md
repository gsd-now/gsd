# Add version subcommand to agent_pool and gsd

## Motivation

When using package manager auto-detection (pnpm dlx, npx, etc.), we need to ensure gsd uses the same version of agent_pool that it was built with. Version mismatches could cause subtle bugs.

Currently there's no way to:
1. Query the version of a binary
2. Ensure gsd and agent_pool versions match when using dlx commands

## Proposed Solution

### 1. Generate version.txt during CI

During CI build, before publishing npm packages, generate a `version.txt` file in the lib folder (NOT inside artifacts) containing the version being published.

**In `.github/workflows/ci.yml`, add to the publish job:**

```yaml
- name: Write version files
  run: |
    VERSION="${GITHUB_REF#refs/tags/v}"
    if [[ ! "$GITHUB_REF" =~ ^refs/tags/v ]]; then
      VERSION="0.0.0-main-${GITHUB_SHA::8}"
    fi
    echo "$VERSION" > libs/agent_pool/version.txt
    echo "$VERSION" > libs/gsd/version.txt
```

### 2. Embed version in binary at compile time

Use a build script to read version.txt and embed it via environment variable.

**Create `crates/agent_pool_cli/build.rs`:**

```rust
fn main() {
    // version.txt is at the lib root, one level up from artifacts/
    // Binary is at libs/agent_pool/artifacts/<platform>/agent_pool
    // So relative to the crate during CI build: ../../libs/agent_pool/version.txt
    let version_path = "../../libs/agent_pool/version.txt";

    if let Ok(version) = std::fs::read_to_string(version_path) {
        println!("cargo:rustc-env=AGENT_POOL_VERSION={}", version.trim());
    } else {
        // No version.txt = development build
        println!("cargo:rustc-env=AGENT_POOL_VERSION=unknown");
    }
}
```

### 3. Add version subcommand

**In `crates/agent_pool_cli/src/main.rs`:**

```rust
const VERSION: &str = env!("AGENT_POOL_VERSION");

#[derive(Debug, Subcommand)]
enum Command {
    /// Print version information
    Version {
        /// Output as JSON (for programmatic access)
        #[arg(long)]
        json: bool,
    },
    // ... other commands
}

// In main():
Command::Version { json } => {
    if json {
        println!(r#"{{"version": "{}"}}"#, VERSION);
    } else {
        println!("{}", VERSION);
    }
    Ok(ExitCode::SUCCESS)
}
```

Output:
```bash
$ agent_pool version
0.1.0

$ agent_pool version --json
{"version": "0.1.0"}

# Development build (no version.txt):
$ agent_pool version
unknown
```

### 4. GSD uses matching version with dlx

When gsd needs to invoke agent_pool via dlx, it should request the same version:

```rust
// In resolve_agent_pool_invocation()
let gsd_version = env!("GSD_VERSION");

// Only pin version if it's a real version (not "unknown")
let package = if gsd_version == "unknown" {
    "@gsd-now/agent-pool".to_string()
} else {
    format!("@gsd-now/agent-pool@{}", gsd_version)
};

AgentPoolInvocation::PackageManager {
    program: "pnpm".to_string(),
    prefix_args: vec!["dlx".to_string(), package],
}
```

This ensures that `gsd@0.1.0` always uses `agent_pool@0.1.0`, not whatever `latest` happens to be.

## Version Format

- Tagged releases: `0.1.0`, `1.2.3`
- Main branch builds: `0.0.0-main-abc12345`
- Development builds: `unknown`

Both gsd and agent_pool will use the same format.

## Testing

1. `agent_pool version` prints version (or "unknown" in dev)
2. `agent_pool version --json` prints JSON
3. Version matches between gsd and agent_pool when using dlx
4. Development builds (no version.txt) print "unknown"
